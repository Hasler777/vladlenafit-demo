# Лендинг «Камилла Тимошенко — врач-диетолог, к.м.н.»

Чистый статический сайт (HTML/CSS/JS, без сборки и зависимостей). Mobile-first.
Сделан по ТЗ `TZ_site_Kamilla_Timoshenko.md`. Каркас взят по аналогии с vladlenafit.ru,
но бизнес-логика другая: сквозное целевое действие — **заявка на контакт**, не оплата.

## Структура

```
kamilla-site/
├── index.html        — лендинг (все блоки ТЗ)
├── privacy.html      — Политика обработки ПД (152-ФЗ) — заглушка под текст заказчика
├── oferta.html       — Договор оферты — заглушка под текст заказчика
├── thanks.html       — страница «Спасибо» (если форму вести на отдельный URL)
├── css/styles.css    — дизайн-система (палитра, типографика, компоненты)
├── js/app.js         — меню, плавный скролл, FAQ, карусель, валидация формы, «наверх»
└── assets/           — фото и файлы (см. ниже)
```

## Дизайн-система

Визуальный язык **воспроизведён по vladlenafit.ru** (по решению клиента — тот же «вайб»):
энергичный «фитнес-стартап», чистый, с лёгкой «айтишностью».

- **Палитра:** чёрный `#161616` + белый + неоновый лайм `#C4F44A` (акцент, hover `#B7FF00`);
  серые секции `#E9F0F3`, бледная плашка `#E3F2BD`.
- **Шрифты:** **Roboto Condensed** (заголовки капсом, узкие, жирные) + **Inter** (текст). Полная кириллица.
- **Фишки:** вырезанный «спрайт»-герой поверх гигантского вордмарка, лаймовые пилюли-бейджи,
  круглые бейджи, белые карточки-модули, лаймовые/тёмные пилюли-CTA.
- Всё управляется CSS-переменными в `:root` — цвета/радиусы/отступы меняются в одном месте
  (оттенки можно подвинуть позже, не трогая разметку).

## Что нужно подставить (плейсхолдеры)

В `assets/` положить и проверить пути:

- `kamilla-hero.png` — фото Камиллы для первого экрана, **вырезанное с фона** (PNG с прозрачностью,
  как спрайт). Ставится поверх вордмарка.
- `kamilla-about.png` — личное фото для блока «Обо мне», тоже **вырезанное с фона** (PNG).
- `kniga-blagodarnosti.pdf` — «Книга благодарности» (PDF уже есть у заказчика).
- `og-preview.jpg` — превью для репостов (Open Graph), ~1200×630.
- Фото кейсов «до/после» — вставить в карточки в секции `#cases` вместо плейсхолдер-иконок
  (**только после подтверждения согласия клиенток на публикацию**).

> Пока фото нет — на его месте аккуратный плейсхолдер-блок с подписью (`onerror` прячет битый `<img>`).
> Вырезку с фона можно сделать любым background-remover'ом; я тоже могу вырезать, если пришлёшь исходник.

## Что подключить перед запуском (бэкенд/аналитика)

1. **Форма** (`js/app.js`, функция submit): сейчас — заглушка с имитацией успеха.
   Вставить реальный `fetch` на эндпоинт заказчика (почта / Telegram-бот).
   Сообщение «Спасибо, Валентина свяжется с вами» уже выводится.
2. **Яндекс.Метрика:** вставить счётчик и задать `window.YM_ID = <номер>` (см. комментарий
   внизу `index.html`). Цели `form_submit` и `telegram_click` уже отправляются из JS.
3. **Реквизиты ИП** (ИНН, ОГРН, ФИО) — в футере и в `privacy.html` / `oferta.html`.
4. **Тексты** политики ПД и оферты — вставить в страницы-заглушки.

