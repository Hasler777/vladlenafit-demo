# Лендинг «Камилла Тимошенко — врач-диетолог, к.м.н.»

Чистый статический сайт (HTML/CSS/JS, без сборки и зависимостей). Mobile-first.
Сделан по ТЗ `TZ_site_Kamilla_Timoshenko.md`. Каркас взят по аналогии с vladlenafit.ru,
но бизнес-логика другая: сквозное целевое действие — **заявка на контакт**, не оплата.

## Структура

```
kamilla-site/
├── index.html        — лендинг (все блоки ТЗ)
├── privacy.html      — Политика обработки ПД (152-ФЗ) — заглушка под текст заказчика
├── oferta.html       — Договор оферты — заглушка под текст заказчика
├── thanks.html       — страница «Спасибо» (если форму вести на отдельный URL)
├── css/styles.css    — дизайн-система (палитра, типографика, компоненты)
├── js/app.js         — меню, плавный скролл, FAQ, карусель, валидация формы, «наверх»
└── assets/           — фото и файлы (см. ниже)
```

## Дизайн-система (v2 — по TZ_site_Kamilla_v2.md, Часть B)

Тёплый «медицинский, но человеческий» образ из фирстиля «Книги благодарности». **Без неона.**

- **Палитра:** крем `--bg #F5F1E8` / `--bg-alt #FBF9F3`, олива `--olive #6E7B4F` / `--olive-deep #4F5938`,
  редкий терракотовый акцент `--accent #C8732E` (только кнопки/цена/ключевые акценты),
  текст `--text #2E2C26`, линии `--line #E2DCCD`, маркер `--mark #CBD79B`.
- **Шрифты:** **Playfair Display** (заголовки H1–H2, антиква) + **Inter** (текст, ≥18px). Кириллица.
- **Ритм:** единый отступ секций 96/56px, чередование фонов, 3 тёмные секции-якоря
  (Энергия, Личка, Финал), личка — кульминация (олива + терракот).
- **Signature:** маркерное выделение `<span class="hl">…</span>` ключевых фраз (1 на экран).
- Всё на CSS-переменных в `:root` — оттенки можно двигать в одном месте.

> Версии: lime-версия (по vladlenafit) и первая тёплая — в git-ветках репозитория Pages
> (`kamilla-v2-lime-backup`, `kamilla-v1-clean-backup`).

## Что нужно подставить (плейсхолдеры)

Фото **есть на руках у заказчика** — положить в `assets/` и проверить пути:

- `kamilla-hero.jpg` — тёплое живое фото Камиллы для hero (справа, формат ~4:5).
- `kamilla-about.jpg` — личное фото для блока «Обо мне» (слева, ~4:5).
- Фото кейсов «до/после» (6 шт., формат 2:3) — вставить в карточки `#cases` вместо плейсхолдер-иконок
  (**после подтверждения согласия клиенток на публикацию**).
- `kniga-blagodarnosti.pdf` — «Книга благодарности» (уже есть).
- `og-preview.jpg` — превью для репостов (Open Graph), ~1200×630.

> Пока фото нет — на его месте аккуратный плейсхолдер с подписью (`onerror` прячет битый `<img>`).
> Картинки рекомендуется сжать в WebP, ниже первого экрана — `loading="lazy"`.

## Что подключить перед запуском (бэкенд/аналитика)

1. **Форма** (`js/app.js`, функция submit): сейчас — заглушка с имитацией успеха.
   Вставить реальный `fetch` на эндпоинт заказчика (почта / Telegram-бот).
   Сообщение «Спасибо, Валентина свяжется с вами» уже выводится.
2. **Яндекс.Метрика:** вставить счётчик и задать `window.YM_ID = <номер>` (см. комментарий
   внизу `index.html`). Цели `form_submit` и `telegram_click` уже отправляются из JS.
3. **Реквизиты ИП** (ИНН, ОГРН, ФИО) — в футере и в `privacy.html` / `oferta.html`.
4. **Тексты** политики ПД и оферты — вставить в страницы-заглушки.
